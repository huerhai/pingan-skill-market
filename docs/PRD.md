## 1. 项目概述

### 1.1 项目背景
平安健康险基于OpenClaw平台进行二次开发，建设自有技能市场，推动保险行业智能化转型。通过技能市场，公司内部及合作伙伴可以共享、使用和开发AI技能，提升业务效率和服务质量。

### 1.2 项目目标
1. 建立中国领先的保险行业AI技能共享平台
2. 促进公司内部AI技能复用和标准化
3. 降低AI技能开发门槛，鼓励创新
4. 建立技能审核和管理机制，确保质量
5. 为未来商业化运营奠定基础

### 1.3 目标用户
1. **保险顾问** - 使用技能提升工作效率
2. **核保人员** - 使用技能进行风险评估
3. **开发人员** - 上传和分享自研技能
4. **管理人员** - 审核和管理技能
5. **合作伙伴** - 使用平台技能服务

## 2. 功能需求

### 2.1 前台功能模块

#### 2.1.1 首页模块
- 搜索功能：支持技能名称、分类、关键词搜索
- 分类展示：6大分类（健康管理、理赔处理、风险分析、客户服务、流程自动化、安全合规）
- 热门推荐：根据下载量、评分、更新时间推荐
- 用户入口：注册、登录、个人中心

#### 2.1.2 技能市场模块
- 技能列表：分页展示，支持排序（最新、最热、评分）
- 技能筛选：按分类、标签、评分筛选
- 技能详情：完整技能信息展示
- 下载功能：支持技能包下载
- 评价系统：用户评分和评论

#### 2.1.3 技能上传模块
- 技能信息填写：中英文名称、描述、分类、标签等
- 文件上传：技能包文件、图标、文档
- 案例说明：使用场景和操作步骤
- 提交审核：提交后进入审核流程

#### 2.1.4 用户中心模块
- 个人信息管理
- 我的下载记录
- 我的收藏技能
- 我的上传记录
- 消息通知

### 2.2 后台管理模块

#### 2.2.1 技能管理
- 技能列表：查看所有技能
- 审核功能：通过、拒绝、需要修改
- 编辑功能：修改技能信息
- 删除功能：删除违规技能
- 内测管理：设置内测人员名单

#### 2.2.2 用户管理
- 用户列表：查看注册用户
- 权限管理：分配不同角色权限
- 行为监控：记录用户操作日志
- 封禁管理：处理违规用户

#### 2.2.3 数据统计
- 下载统计：按时间、技能、用户统计
- 用户活跃度：日活、月活、留存率
- 技能评分：平均分、评价数量
- 系统监控：服务器状态、API调用

#### 2.2.4 系统设置
- 分类管理：添加、修改、删除分类
- 标签管理：管理技能标签
- 公告管理：发布平台公告
- 权限配置：配置角色权限

### 2.3 技能详情要求

#### 2.3.1 基本信息
- 中文名称（唯一）
- 英文名称
- 技能描述（详细功能说明）
- 使用案例说明（具体使用场景）
- 技术规格（开发平台、语言、框架等）

#### 2.3.2 元数据
- 分类：至少选择一个分类
- 标签：最多5个标签
- 版本号：遵循语义化版本
- 兼容性：支持的OpenClaw版本
- 文件大小：技能包大小

#### 2.3.3 开发者信息
- 开发者姓名/团队
- 联系方式
- 开发经验介绍
- 其他技能链接

## 3. 非功能需求

### 3.1 性能要求
- 页面加载时间：首屏加载 < 3秒
- API响应时间：< 500ms
- 并发支持：支持1000+并发用户
- 文件上传：支持最大500MB文件

### 3.2 安全要求
- 用户认证：JWT Token认证
- 数据加密：敏感数据加密存储
- 权限控制：RBAC权限模型
- 防攻击：SQL注入、XSS防护
- 审计日志：完整操作日志记录

### 3.3 可用性要求
- 响应式设计：支持PC、平板、手机
- 浏览器兼容：Chrome、Firefox、Safari、Edge
- 无障碍访问：符合WCAG 2.1标准
- 错误处理：友好的错误提示

### 3.4 可维护性要求
- 代码规范：遵循公司编码规范
- 文档完整：API文档、部署文档
- 监控告警：系统监控和自动告警
- 备份恢复：数据定期备份

## 4. 业务流程

### 4.1 技能上传流程
```
用户登录 → 进入上传页面 → 填写技能信息 → 上传文件 → 提交审核 → 等待审核 → 审核通过/拒绝
```

### 4.2 技能审核流程
```
管理员登录 → 查看待审核列表 → 查看技能详情 → 填写审核意见 → 选择审核结果 → 通知用户
```

### 4.3 技能下载流程
```
用户登录 → 浏览技能市场 → 查看技能详情 → 点击下载 → 记录下载历史 → 使用技能
```

## 5. 数据模型

### 5.1 核心数据表

#### 5.1.1 技能表 (skills)
```sql
id: 主键
name_zh: 中文名称 (唯一)
name_en: 英文名称
description: 技能描述
category_id: 分类ID
tags: 标签数组
version: 版本号
file_url: 文件地址
file_size: 文件大小
download_count: 下载次数
rating: 平均评分
review_count: 评价数量
status: 状态 (draft, pending, approved, rejected)
developer_id: 开发者ID
created_at: 创建时间
updated_at: 更新时间
```

#### 5.1.2 用户表 (users)
```sql
id: 主键
username: 用户名
email: 邮箱
password_hash: 密码哈希
role: 角色 (user, developer, admin)
avatar: 头像
company: 公司
department: 部门
created_at: 创建时间
last_login: 最后登录时间
```

#### 5.1.3 审核记录表 (reviews)
```sql
id: 主键
skill_id: 技能ID
reviewer_id: 审核员ID
status: 审核状态
comments: 审核意见
internal_test_users: 内测人员名单
reviewed_at: 审核时间
```

## 6. API设计

### 6.1 公共API

#### 6.1.1 技能相关
- `GET /api/v1/skills` - 获取技能列表
- `GET /api/v1/skills/:id` - 获取技能详情
- `POST /api/v1/skills` - 创建技能（需要登录）
- `PUT /api/v1/skills/:id` - 更新技能（需要权限）
- `POST /api/v1/skills/:id/download` - 下载技能

#### 6.1.2 用户相关
- `POST /api/v1/auth/register` - 用户注册
- `POST /api/v1/auth/login` - 用户登录
- `GET /api/v1/users/profile` - 获取用户信息
- `PUT /api/v1/users/profile` - 更新用户信息

### 6.2 管理API

#### 6.2.1 审核相关
- `GET /api/v1/admin/skills/pending` - 获取待审核技能
- `POST /api/v1/admin/skills/:id/review` - 提交审核结果
- `GET /api/v1/admin/stats` - 获取统计信息

## 7. 界面设计

### 7.1 设计原则
1. **简洁明了** - 界面简洁，功能明确
2. **保险特色** - 体现保险行业专业性
3. **用户友好** - 操作简单，提示清晰
4. **响应式** - 适配各种设备

### 7.2 色彩方案
- 主色调：蓝色 (#1a73e8) - 代表专业、信任
- 辅助色：橙色 (#ff6b35) - 代表活力、创新
- 背景色：浅灰色 (#f8f9fa) - 舒适阅读
- 文字色：深灰色 (#333) - 清晰易读

### 7.3 字体方案
- 主要字体：Segoe UI, Microsoft YaHei
- 字号规范：14px基础，层次分明
- 行高规范：1.6倍行距，舒适阅读

## 9. 风险评估

### 9.1 技术风险
- OpenClaw平台兼容性问题
- 大文件上传性能问题
- 高并发访问压力

### 9.2 业务风险
- 技能质量参差不齐
- 用户活跃度不足
- 版权和合规问题

### 9.3 应对措施
- 技术预研和原型验证
- 建立严格的审核机制
- 制定用户激励计划
- 完善法律合规审查
